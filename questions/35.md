# Процедуры улучшения качества данных. 5 методов улучшения качества данных

## 1. Стандартизация (Standardization)

**Приведение данных к единому формату для обеспечения согласованности.**

###  Зачем?
Чтобы данные в разных источниках или столбцах выглядели одинаково — и их можно было сравнивать, объединять, анализировать.

###  Пример:
- Телефоны: `89151234567`, `+7 (915) 123-45-67`, `915 123 45 67`  
  → Единый формат: `+7 (915) 123-45-67`

###  Цель:
Устранить дублирование и несогласованность.

---

## 2. Извлечение (Extraction / Parsing)

**Выделение структурированных данных из неструктурированного текста.**

###  Зачем?
Часто данные хранятся в виде строк — например, описание товара, техническая спецификация, адрес. Нужно **вытащить из них отдельные поля**.

###  Пример:
Из строки:  
`"ОТВОД 15 – 273X20 – R1370"`  
→ Извлекаются параметры:  
- Диаметр трубы: `273`  
- Толщина стенки: `20`  
- Радиус изгиба: `1370`

###  Цель:
Превратить «текст» в «структурированные данные», пригодные для анализа.

---

## 3. Идентификационный анализ (Identification Analysis)

**Определение типа данных в столбце** — что это: телефон, email, ФИО, адрес и т.д.

###  Зачем?
Чтобы правильно обработать данные:
- Если столбец содержит телефоны → применить стандартизацию
- Если email → проверить валидность

###  Пример:
Столбец содержит значения:  
`+7 916 123 45 67`, `ivan@domain.ru`, `Петров А.С.`  
→ Система определяет тип каждого значения.

###  Цель:
Автоматизировать обработку данных — зная тип, можно применить нужную логику.

---

## 4. Поиск совпадений (Matching / Deduplication)

**Поиск и объединение дубликатов** — одинаковых записей, представленных по-разному.

###  Зачем?
Чтобы:
- Не считать одного клиента дважды
- Не отправлять два письма одному человеку
- Не дублировать записи в базе

###  Пример:
- `"Кирпич красный"`  
- `"Красный кирпич"`  
- `"Кр.Кирпич"`  
→ Все три записи относятся к одному материалу → объединяются.

###  Цель:
Устранить дублирование и повысить точность анализа.

---

## 5. «Золотая запись» (Golden Record / Master Data)

**Создание единой, самой полной и точной версии записи из нескольких источников.**

###  Зачем?
Когда данные об одном объекте (клиент, продукт, поставщик) хранятся в разных системах — нужно создать **единую «золотую» версию**, которая будет использоваться всеми.

###  Пример:
| Источник | Данные |
|--------|--------|
| CRM | Имя: Иван, Телефон: +7... |
| Доставка | Email: ivan@mail.ru, Город: Москва |
| Бухгалтерия | Дата регистрации: 2025-01-01 |

→ **Золотая запись**:  
- Имя: Иван  
- Телефон: +7...  
- Email: ivan@mail.ru  
- Город: Москва  
- Дата регистрации: 2025-01-01

###  Цель:
Создать **единый источник правды** — чтобы все отделы работали с одними и теми же данными.
