# **Целевой вид данных. Преобразование данных. Методы преобразования. Нормализация данных**

## Целевой вид данных

Целевой вид данных — это формат или структура, в которую нужно преобразовать исходные данные для конкретной задачи анализа или загрузки в систему. Например, для машинного обучения целевой вид — числовые признаки в определённом диапазоне; для базы данных — соответствие схеме таблицы.

## Преобразование данных

Преобразование (трансформация) — процесс изменения данных из исходного формата в целевой для совместимости с системой, улучшения качества или подготовки к анализу. Включает очистку, изменение типов, форматов и структуры.​

### Основные цели преобразования:

   * Обобщение больших массивов в понятные параметры

   * Приведение к единому формату (консолидация)

   * Повышение качества данных

   * Соответствие требованиям модели/системы

| Метод                | Описание                            | Пример                              |
| -------------------- | ----------------------------------- | ----------------------------------- |
| Преобразование типов | Изменение формата данных            | "Да"/"Нет" → true/false astera​     |
| Фильтрация           | Выбор подмножества по критериям     | Только клиенты >18 лет              |
| Группировка          | Объединение по признаку             | Продажи по месяцам                  |
| Сортировка           | Упорядочивание                      | По дате, по алфавиту                |
| Слияние              | Объединение таблиц                  | JOIN по ID клиента                  |
| Обогащение           | Добавление информации               | Геолокация по IP-адресу             |
| Биннинг              | Группировка непрерывных в категории | Возраст → "молодой/средний/старший" |

## Нормализация данных

Нормализация — приведение данных к единой шкале, чтобы признаки с разными диапазонами стали сопоставимы. Критически важна для ML-алгоритмов (kNN, нейросети), где большие значения доминируют.​

### Основные методы нормализации:

1. Min-Max (масштабирование в ):​
x′=x−xminxmax−xminx′=xmax−xminx−xmin

    Пример: рост см → 160 см превращается в (160-150)/(200-150) = 0.2

1. Z-Score (стандартизация):
z=x−xˉσz=σx−xˉ

    Результат: среднее = 0, σ = 1

    Пример: рост 170 см при среднем 175 и σ=10 → z = (170-175)/10 = -0.5

1. Десятичное масштабирование:
x′=x10kx′=10kx

    k — число знаков в max значении

    Пример: 450 → 0.45 (k=3)
