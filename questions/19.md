# Большие данные. Критерии "больших данных". 3V+
Термин "Большие данные" описывает не просто огромные массивы информации, а совокупность технологий и методов их обработки, которые позволяют извлекать ценность из данных, недоступных для традиционных СУБД

## Критерии "Больших данных" (концепция V-модели)
Первоначально концепция строилась на «3V», предложенных аналитиком Дугом Лани еще в начале 2000-х. Однако с развитием технологий в 2020-х годах модель расширилась до 5V

#### 1. Volume (объем)
Ключевой фактор. Речь идет о физическом объеме данных 
*   Суть - данных так много, что они не помещаются на одном сервере

#### 2. Velocity (скорость)
Скорость, с которой данные генерируются и обрабатываются 
* Суть - это критично для IoT (интернета вещей), финансовых транзакций и соцсетей. Данные поступают непрерывно, поэтому анализ должен идти в реальном времени

#### 3. Variety (разнообразие)
Данные поступают в разных форматах:
*   Структурированные - таблицы SQL
*   Полуструктурированные - JSON, XML
*   Неструктурированные - текст сообщений, видео, аудиозаписи, логи

## Расширенные критерии

#### 4. Veracity (доверенность)
Насколько данным можно доверять?
*   Суть - очистка данных от шума, фейков, дипфейков и ошибок показаний. Без достоверности объем данных становится бесполезным и даже опасным для принятия решений

#### 5. Value (ценность)
Главный критерий для бизнеса
*   Суть - данные не имеют смысла, если их хранение стоит дороже, чем польза, которую они приносят
